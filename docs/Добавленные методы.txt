Добавленные методы для работы со сканером и онлайн-кассой:

1. scaner_on. Метод включения сканера на считывание кода.

	Пример запроса  «scaner_on»:
	
	http://localhost:8888/devices?method=scaner_on
	
	Положительный ответ может быть получен, если  сессия была открыта, а сканер выключен
	<Response Result="1" Error="0" Message="Ok" />

	Ошибочный ответ может быть получен, если нет открытой сессии, сканер уже включен, или сканер не отвечает.

	<Response Result="1" Error="1" Message="Сканер уже включен" />
			
	ПРИМЕЧАНИЕ: После считывания кода, сканер будет переведен в состояние "выключен". Если нужно считать другой код,
	сканер нужно включить повторно методом scaner_on
	
2.	get_status_scaner. Метод получения статуса сканера и полседнего считанного в сессии кода.
		
	Пример запроса статуса «get_status_scaner»:

	http://localhost:8888/devices?method=get_status_scaner
	
	<Response Result="1" Error="0" Message="Ok">	
		<Scaner Status="0" IsStarted="False" IsError="False" LastCode="12341562"/>	
	</Response>

	Status – Код состояния устройства. 0 – ошибок нет, >=200 – ошибочное состояние устройства при 
	котором устройство не может функционировать.
	IsError – логическое (bool) значение находится ли устройство в состоянии ошибки
	IsStarted – логическое (bool) значение, показывает включен ли сканер на чтение кода
	LastCode - последний считанный в этой сессии сканером код


3. scaner_off. Метод выключения сканера на считывание кода.

	Пример запроса  «scaner_off»:
	
	http://localhost:8888/devices?method=scaner_off
	
	Положительный ответ может быть получен, если  сессия была открыта, а сканер включен
	<Response Result="1" Error="0" Message="Ok" />

	Ошибочный ответ может быть получен, если нет открытой сессии, сканер выключен или не отвечает.

	<Response Result="1" Error="1" Message="Сканер выключен" />
			
	ПРИМЕЧАНИЕ: После считывания кода, сканер будет переведен в состояние "выключен" автоматически. Данный метод следует вызывать
	только для ручного выключения (например при отмене операции).
	
	
4. printer_print. Метод печати чека на фискальном принтере.
		
	ПРИМЕЧАНИЕ: В отличие от остальных методов, данный должен быть выполнен методом POST, и содержать тело в формате JSON 
	
	Пример запроса статуса «get_status_printer»:

	http://localhost:8888/devices?method=get_status_printer

	ТЕЛО ЗАПРОСА:

	{"type": "sell", "items": [
								{"type": "text", "text": " СТРОКА ДЛЯ ПЕЧАТИ ДО ФИСКАЛЬНОЙ ЧАСТИ"},
								{"type": "position", "name": "Наименование товара1", "price": "1000", "quantity": "1", "paymentObject": "service/commodity"},		
								{"type": "position", "name": "Наименование товара2", "price": "100", "quantity": "10", "paymentObject": "service/commodity"},								
								{"type": "text", "text": " СТРОКА ДЛЯ ПЕЧАТИ ПОСЛЕ ФИСКАЛЬНОЙ ЧАСТИ"}
							  ], "payments": [{"type": "electronically", "sum": "200"},{"type": "cash", "sum": "1800"}]}
							  


	для фискальных позиций указывается "type": "position". Чек будет открыт после первой позиции. Последующие нефискальные строки ("type": "text")
	будут напечатаны после закрытия чека
    поле price - цена за единицу товара\услуги в копейках
	поле quantity - кол-во единиц товара
	поле paymentObject для позиции чека может принимать значения service(Услуга) или commodity(Товар)
	
	для закрытия чека используется поле payments.
	Может содержать данные о платежах 
	"type": "electronically" - платеж картой
	"type": "cash" - платеж наличными
	Могут быть использованы как оба способа, так и только один. Главное условие - должна совпадать сумма платежей с суммой строк табличной части
	"sum": "1800" - сумма платежа в копейках
	
	Положительный ответ может быть получен, если принтер функционирует и не занят, и правильно заполнено тело запроса
	<Response Result="1" Error="0" Message="Ok" />

	Ошибочный ответ может быть получен, если принтер занят, неисправен, или ошибка в теле запроса.

	<Response Result="1" Error="1" Message="Принтер занят" />
						

5.	get_status_printer. Метод получения статуса принтера.
		
	Пример запроса статуса «get_status_printer»:

	http://localhost:8888/devices?method=get_status_printer
	
	<Response Result="1" Error="0" Message="Ok">	
		<Printer Status="0" IsStarted="False" IsBusy="True" IsError="False""/>	
	</Response>

	Status – Код состояния устройства. 0 – ошибок нет, >=200 – ошибочное состояние устройства при 
	котором устройство не может функционировать.
	IsError – логическое (bool) значение находится ли устройство в состоянии ошибки					
	IsBusy - логическое (bool) значение показывает, что устройство занято процессом	печати
